# Azure Blob Storage – Photo Upload via SAS

This document explains how to set up Azure Blob Storage for the TrampoHero
photo-upload flow and how to configure the backend accordingly.

---

## Overview

The upload flow avoids sending files through the Express server:

```
Client                   Backend                  Azure Blob Storage
  |                         |                            |
  |-- POST /api/uploads/sas -->                          |
  |                    generates SAS URL                 |
  |<-- { uploadId, sasUrl } --                           |
  |                                                      |
  |-- PUT <sasUrl> (file bytes) -----------------------> |
  |                                                      |
  |-- POST /api/uploads/confirm (uploadId) -->           |
  |                    verifies blob exists              |
  |                    stores permanent URL              |
  |<-- { url } --                                        |
```

---

## 1. Create an Azure Storage Account

```bash
# Login
az login

# Create resource group (skip if one already exists)
az group create --name trampohero-rg --location brazilsouth

# Create storage account (name must be globally unique, 3-24 lowercase chars)
az storage account create \
  --name trampoherofiles \
  --resource-group trampohero-rg \
  --location brazilsouth \
  --sku Standard_LRS \
  --kind StorageV2 \
  --allow-blob-public-access false
```

## 2. Create the Uploads Container

```bash
ACCOUNT_KEY=$(az storage account keys list \
  --account-name trampoherofiles \
  --resource-group trampohero-rg \
  --query "[0].value" -o tsv)

az storage container create \
  --name uploads \
  --account-name trampoherofiles \
  --account-key "$ACCOUNT_KEY" \
  --public-access off
```

> **Security note:** The container is private. Objects are only accessible via
> the time-limited SAS URLs generated by the backend.

## 3. Retrieve the Account Key

```bash
az storage account keys list \
  --account-name trampoherofiles \
  --resource-group trampohero-rg \
  --query "[0].value" -o tsv
```

## 4. Set Backend Environment Variables

Add the following to `backend/.env` (see `backend/.env.example`):

```env
AZURE_STORAGE_ACCOUNT_NAME=trampoherofiles
AZURE_STORAGE_ACCOUNT_KEY=<key from step 3>
AZURE_STORAGE_CONTAINER_NAME=uploads
```

If these variables are absent the upload endpoints return `503 Service
Unavailable`, so the rest of the API continues to work.

## 5. Set Env Vars in Azure Container Apps

```bash
az containerapp update \
  --name trampohero-api \
  --resource-group trampohero-rg \
  --set-env-vars \
    AZURE_STORAGE_ACCOUNT_NAME=trampoherofiles \
    AZURE_STORAGE_ACCOUNT_KEY=secretref:azure-storage-key \
    AZURE_STORAGE_CONTAINER_NAME=uploads
```

For secrets, store them in Container Apps secrets first:

```bash
az containerapp secret set \
  --name trampohero-api \
  --resource-group trampohero-rg \
  --secrets "azure-storage-key=<key>"
```

## 6. Manual Test (curl)

> Replace `TOKEN` with a valid JWT obtained from `POST /api/auth/login`.

### Step 1 – Request a SAS URL

```bash
TOKEN="<your_jwt_token>"

curl -s -X POST http://localhost:5000/api/uploads/sas \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"contentType":"image/jpeg"}' | tee /tmp/sas.json
```

Expected response:

```json
{
  "success": true,
  "data": {
    "uploadId": "6650abc123...",
    "sasUrl": "https://trampoherofiles.blob.core.windows.net/uploads/users/<userId>/<timestamp>.jpeg?sv=...&sig=...",
    "blobName": "users/<userId>/<timestamp>.jpeg",
    "expiresInMinutes": 15
  }
}
```

### Step 2 – Upload the File Directly to Azure

```bash
SAS_URL=$(jq -r '.data.sasUrl' /tmp/sas.json)

curl -s -X PUT "$SAS_URL" \
  -H "x-ms-blob-type: BlockBlob" \
  -H "Content-Type: image/jpeg" \
  --data-binary @/path/to/photo.jpg
```

A successful upload returns HTTP `201 Created` with an empty body.

### Step 3 – Confirm the Upload

```bash
UPLOAD_ID=$(jq -r '.data.uploadId' /tmp/sas.json)

curl -s -X POST http://localhost:5000/api/uploads/confirm \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"uploadId\":\"$UPLOAD_ID\"}"
```

Expected response:

```json
{
  "success": true,
  "data": {
    "url": "https://trampoherofiles.blob.core.windows.net/uploads/users/<userId>/<timestamp>.jpeg",
    "upload": { ... }
  }
}
```

### Optional – Associate with a Job

Pass `jobId` and/or `checkinId` in the SAS request body:

```bash
curl -s -X POST http://localhost:5000/api/uploads/sas \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"contentType":"image/png","jobId":"<mongoId>","checkinId":"abc123"}'
```

## 7. Allowed Content Types

Only the following MIME types are accepted:

| MIME type    | Extension |
|--------------|-----------|
| `image/jpeg` | `.jpeg`   |
| `image/png`  | `.png`    |
| `image/webp` | `.webp`   |

Requests with any other `contentType` will be rejected with `400 Bad Request`.

## 8. SAS Token Parameters

| Parameter       | Value                   |
|-----------------|-------------------------|
| Permissions     | `cw` (create + write)   |
| Expiry          | 15 minutes from issue   |
| Allowed methods | PUT only (no GET/DELETE) |
